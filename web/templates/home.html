The Problem

A parent with three doctors and no single source of truth

Prescriptions arrive as photos

Handwritten prescriptions forwarded on WhatsApp. No version history. No way to know if a dosage changed.

Lab reports live in three apps

Each hospital has its own portal. Trend tracking across visits requires manual comparison across PDFs.

No doctor sees the full list

The cardiologist does not know what the endocrinologist prescribed. Drug interactions get missed — until they cause harm.

Infrastructure

India's open health infrastructure

UHI is India's open protocol for health services -- the same idea as UPI, applied to healthcare. CareCircle is built directly on this stack, which means your parent's records from any ABDM-linked hospital or lab can flow into one place with their consent.

ABHA Health ID

One ID for every Indian

Ayushman Bharat Health Account -- a 14-digit identifier that links a person's health records across all hospitals, labs, and pharmacies. 847 million IDs created. Your parent's Apollo visit, the local clinic prescription, and the diagnostic report are all accessible through one number.

UHI Unified Health Interface

Open protocol, not a closed app

UHI is the open network layer that lets any compliant app discover doctors, book consultations, and access health records -- without being locked into one hospital's portal. Governed by the National Health Authority. Think of it as the routing layer beneath every health interaction.

HIE-CM Consent Manager

Patient controls every access

Before any record moves, the patient approves the request -- what data, from which provider, to whom. The Health Information Exchange and Consent Manager is the layer that enforces this. CareCircle uses it to ensure family members see only what the patient has explicitly permitted.

Intelligence

Two-tier processing, one coherent picture

Raw inputs go through a fast extraction layer first. Structured data then reaches the reasoning layer. This keeps the system accurate and cost-efficient.

TIER 1 Gemini Flash ~80% of work
  • + OCR on prescription photos
  • + PDF lab report extraction
  • + Voice note transcription
  • + Hindi, English, and Hinglish detection
  • + Structured JSON output for each input type
TIER 2 Claude Sonnet ~20% of work
  • + Drug interaction analysis across all active medications
  • + Daily plain-language morning briefing
  • + Crisis detection and response
  • + Appointment gap detection
  • + Deviation detection (wrong timing, missed dose)
Raw input never reaches the reasoning layer. Gemini parses first, Claude reasons over structured JSON. This keeps the reasoning layer accurate and API costs minimal.

Today V2 Roadmap: blockchain anchoring

ABHA-linked consent and audit trail

Every data access in CareCircle is logged to a consent table, tied to the patient's ABHA ID. The HIE-CM layer ensures no family member or doctor can pull records without an explicit approval event on record. Today this lives in Supabase with full timestamps and role attribution.

Records are versioned and never deleted. When a prescription changes, the old version is marked superseded -- the full history is preserved with its ABHA ID linkage intact. In V2, consent artifacts will be anchored to a distributed ledger for cryptographic tamper-evidence.
